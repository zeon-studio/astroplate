This skill provides a high-level map of the Astro template ecosystem, built on Astro 6.1+, Tailwind CSS v4, and React Islands.


## Data Flow: From Content to Page

This template leverages the Astro Content Collections API:

1. **Content Definition**: A user creates `src/content/blog/my-post.md`.
2. **Validation**: Astro validates the frontmatter against the schema defined in `src/content.config.ts`.
3. **Routing & Fetching**: `src/pages/blog/[single].astro` uses `getCollection('blog')` and `getStaticPaths` to generate the routes dynamically at build time.
4. **Rendering**: The page receives the data, wraps it in `Base.astro`, and renders the HTML. Interactive React components are injected as isolated "Islands" using `client:*` directives.

## What to Edit vs. What to NEVER Touch

### ✅ Safe to Edit

- **`src/content/**`\*\*: Where all text and image content lives.
- **`src/config/**`\*\*: Where global settings, menus, and theming (colors/fonts) are defined.
- **`src/pages/**`\*\*: Add new routes or modify page templates here.
- **`src/layouts/**`\*\*: Modify or add UI components.
- **`src/content.config.ts`**: Edit this if you add new content folders or change frontmatter requirements.

### 🚫 NEVER Touch (Without Extreme Caution)

- **`src/styles/generated-theme.css`**: Auto-generated by scripts. Your edits will be overwritten instantly.
- **`scripts/`**: The build and dev scripts. Modifying `themeGenerator.js` or `jsonGenerator.js` can break the entire CSS pipeline.

## Build Pipeline Architecture

When `pnpm dev` or `pnpm build` is executed, the `package.json` scripts trigger `concurrently` or sequential execution of:

1. `node scripts/themeGenerator.js` -> Compiles `theme.json` to `generated-theme.css`.
2. `node scripts/jsonGenerator.js` -> Often generates a search index (`search.json`) from markdown files.
3. `astro dev` / `astro build` -> The standard Astro build process.
4. `node scripts/llmsGenerator.js` -> (Post-build) generates `llms.txt` for AI agent consumption based on `config.json` rules.
